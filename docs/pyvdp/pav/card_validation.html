Payment Account Validation¶

See also

https://developer.visa.com/products/pav/reference

The Payment Account Validation API allows applications to run validations of the payment account before processing a transaction ensuring greater probability of success and allowing for a more seamless transaction flow.

Request¶

class pyvdp.pav.request.VisaPavRequest(method, http_verb, query_string='', data=None, headers=None)[source]¶

Instantiates connection to PaymentAccountValidation VDP API endpoint.

Parameters:
  • method (str) – Required. VDP method name.
  • http_verb (str) – Required. HTTP verb (GET, POST).
  • query_string (str) – Conditional. Query string to append to URL.
  • data (object) – Conditional. Object to submit as POST/PUT payload.
  • headers (dict) – Optional. Additional headers as dictionary.

Usage¶

pyvdp.pav.cardvalidation.send(data: pyvdp.pav.data.PavTransaction)[source]¶

Submits a payment account validation request.

Parameters:data (PavTransaction) – Required. Instance of PavTransaction().
Returns:A response from VDP.
Example:
from visa.pav.data import PavTransaction
from visa.pav import cardvalidation

data = PavTransaction(stan=123456,
                      pan='1234567812345678',
                      expiry_date='2020-04',
                      cvv2='123')

result = cardvalidation.send(data=data)
